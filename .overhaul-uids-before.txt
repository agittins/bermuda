4. SUMMARY — DISTINCT unique_id FORMAT PATTERNS: 13 total
=====================================================================
  1. {mac}                                            device_tracker, base sensor (area), BermudaEntity base
  2. {mac}_floor                                      sensor
  3. {mac}_scanner                                    sensor
  4. {mac}_rssi                                        sensor
  5. {mac}_range                                       sensor
  6. {mac}_{scanner_wifi_or_ble_mac}_range             sensor (per-scanner)
  7. {mac}_{scanner_wifi_or_ble_mac}_range_raw         sensor (per-scanner)
  8. {mac}_area_switch_reason                          sensor (diagnostic, disabled by default)
  9. {mac}_area_last_seen                              sensor
 10. {mac}_micro_location                              sensor
 11. {mac}_in100_vcc / _in100_temperature / _in100_adc_voltage   sensor (3 variants, IN100-only devices)
 12. {mac}_ref_power                                   number
 13. {mac}_mobility                                    select
  + 5 fixed-literal globals: BERMUDA_GLOBAL_{PROXY_COUNT,ACTIVE_PROXY_COUNT,DEVICE_COUNT,
    VISIBLE_DEVICE_COUNT,NEARBY_DEVICES}
  + special device-address forms feeding {mac}: plain MAC, "{uuid}_{major}_{minor}" (iBeacon),
    32-char IRK hex (private BLE), scanner wifi/BT-registry MAC.

Where "{mac}" = self._device.unique_id, which is itself derived from the (possibly special-form,
see section 1) BermudaDevice.address/beacon_unique_id at device-creation time.

ANOMALY NOTED: entity.py:106-108 (BermudaEntity.unique_id) and sensor_entities.py:42-47
(BermudaSensor.unique_id, which subclasses BermudaEntity) are byte-for-byte identical
overrides (`return self._device.unique_id`). The override in sensor_entities.py is redundant
unless it exists purely for the docstring — worth flagging as dead/duplicate code in the debt
inventory.

Regression coverage: tests/test_unique_id_regression.py (324 lines) exists specifically to pin
every one of the patterns above — any refactor of unique_id logic MUST keep that suite green,
per ARCHITECTURE.md's explicit "(do NOT change without migration)" warning.
